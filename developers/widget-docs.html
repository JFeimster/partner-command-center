Iframe widget

Widget architecture

The widget layer contains a standalone funding-readiness widget and an embed loader. The standalone widget can be opened directly or embedded in an iframe generated by /widgets/embed.js. 

/widgets/
  widget-config.js
  funding-widget.html
  funding-widget.css
  funding-widget.js
  embed.js
  README.md

Direct preview

/widgets/funding-widget.html?partner_id=MS-FB-1024&source=partner-site&theme=dark

Supported query parameters:

ParameterPurpose
partner_idPartner attribution ID shown in the widget and saved with demo inquiries.
sourceSource label for reporting or CRM mapping later.
themedark or light.
titleOverrides the widget headline.
subtitleOverrides the intro copy.
brandOverrides the brand label.
ctaOverrides the submit button label.

Embed snippet

<script
  src="https://YOUR-DOMAIN.com/widgets/embed.js"
  data-moonshine-partner-id="MS-FB-1024"
  data-moonshine-source="partner-site"
  data-moonshine-theme="dark">
</script>

Widget events

The widget emits parent-page events through postMessage.

EventMeaning
moonshine.widget.readyWidget loaded.
moonshine.widget.submittedDemo inquiry saved locally.
moonshine.widget.errorRequired field or consent problem.
moonshine.widget.resizedIframe height changed.

Production upgrade requirements

  • POST inquiries to a real lead submission endpoint.
  • Validate partner IDs server-side.
  • Verify consent and preserve timestamp/source.
  • Add spam protection and rate limiting.
  • Route valid inquiries to CRM or review queue.
  • Add audit logs and error handling.
  • Update privacy policy and terms to match actual data collection.
