Start here

Local run

Because this repo is static, a basic local web server is enough.

cd partner-command-center
python -m http.server 8080

Then open:

http://localhost:8080/index.html
http://localhost:8080/partner-access.html
http://localhost:8080/dashboard.html
http://localhost:8080/admin/index.html

Script and data loading order

For dashboard pages, load data files first, shared scripts second, dashboard modules third, and the page controller last.

<script src="./data/sample-leads.js"></script>
<script src="./scripts/config.js"></script>
<script src="./scripts/storage.js"></script>
<script src="./dashboard/dashboard-config.js"></script>
<script src="./dashboard/dashboard-state.js"></script>
<script src="./dashboard.js"></script>

Manual smoke tests

  • Open the homepage and confirm CTAs route to partner access and dashboard.
  • Create a local partner profile on partner-access.html.
  • Confirm /welcome/index.html displays the partner ID and profile.
  • Open dashboard.html and confirm metrics, leads, resources, marketplace, training, and links render.
  • Create a demo lead, update its status, and export dashboard JSON.
  • Open /tools/funding-readiness-checklist.html and confirm local progress saves.
  • Open /widgets/funding-widget.html with a partner ID query parameter.
  • Open /admin/index.html and confirm tables, filters, export, and local admin notes work.

Static deployment notes

The repo can be deployed as a static site. The current vercel.json may include an auto-deploy gate while the repo is being assembled. Remove or reverse that gate when you intentionally want Vercel builds to run.

If deployment is gated, Git pushes may still appear as deployment attempts in Vercel, but the ignored build command should prevent the build from completing.

Common issues

  • If pages look unstyled, confirm relative paths use ./styles/ at root and ../styles/ in nested folders.
  • If dashboard sections are empty, confirm Batch 04 data files and Batch 08 dashboard files are present.
  • If local data seems stale, clear browser localStorage or use dashboard reset.
  • If widget embed height looks wrong, confirm postMessage events are not blocked by the parent page.
