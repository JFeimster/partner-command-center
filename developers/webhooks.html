Webhook purpose

Webhooks will let external systems notify the Partner Command Center when partner signups, widget inquiries, CRM updates, lead status changes, or document events occur.

In the current repo, webhook files are examples only. No endpoint is live unless a backend is implemented.

Recommended event types

EventSourcePurpose
partner.signup.createdTally or partner formCreate or queue partner profile review.
widget.inquiry.createdEmbeddable widgetCreate a lead inquiry with partner attribution.
lead.status.updatedCRM or funding deskSync lead status into dashboard and event log.
partner.link.clickedTracking serviceLog partner traffic and campaign attribution.

Webhook security requirements

  • Use HTTPS only.
  • Verify signatures for every inbound webhook.
  • Reject unsigned or expired payloads.
  • Validate event type and schema before processing.
  • Make handlers idempotent using event_id.
  • Log accepted, rejected, and duplicate events.
  • Rate-limit event sources and quarantine suspicious payloads.

Expected response

{
  "ok": true,
  "received": true,
  "event_id": "evt_123",
  "message": "Webhook received for processing."
}
