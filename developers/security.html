Security boundary

Current security posture

The current repo is static and does not provide production authentication, authorization, secure server storage, backend validation, audit logs, encrypted databases, or secret management.

Static pages are appropriate for prototypes, docs, mock dashboards, and local demos. They are not appropriate for secure lead processing or admin operations without a backend.

Do not store sensitive data in the static demo

  • No Social Security numbers.
  • No full tax IDs or EIN documents.
  • No bank account numbers.
  • No bank login credentials.
  • No private financial documents.
  • No full loan applications.
  • No underwriting files or provider decisions.

Production controls required before live data

Identity and access

Authentication, role-based access control, partner scoping, admin scoping, and session management.

Server-side validation

Validate every field, event, partner ID, consent flag, source, and status transition server-side.

Secrets and signatures

Use environment variables, webhook signatures, key rotation, and no secrets in static files.

Audit logs

Log creates, updates, status changes, exports, admin reviews, and webhook processing outcomes.

Minimum threat model

  • Fake partner IDs submitting leads through widgets.
  • Spam submissions and bot traffic.
  • Unauthorized partner access to other partners’ leads.
  • Admin screens exposed without authentication.
  • Webhook replay attacks.
  • Malformed payloads changing statuses or polluting CRM records.
  • Leaking sensitive lead data through browser storage or exports.

Static hosting recommendations

  • Keep security headers in vercel.json or host config.
  • Do not publish real API keys, tokens, CRM secrets, or webhook secrets in the repo.
  • Use clear labels for static demo/admin pages.
  • Block or discourage indexing of admin and example API docs where appropriate.
  • Add production access controls before any real data appears.

Incident readiness checklist

  • Know how to disable forms, widgets, and webhooks quickly.
  • Keep partner IDs revocable.
  • Keep webhook secrets rotatable.
  • Log and review rejected submissions.
  • Document data retention and deletion process.
