Privacy basics

Partner workspace now. Real privacy work before real data.

This page explains how the static Partner Command Center uses browser-partner workspace state and what future backend versions must handle before live partner or lead data is collected.

Partner Access View Terms
Do not enter sensitive data

Local browser storage

The partner workspace may store profile, lead, note, training, attribution, and analytics-like workspace data in your browser. It is not secure backend storage.

Scope of this privacy page

This privacy page applies to the static Moonshine Partner Command Center demo files in this repo. It is a practical privacy overview, not a final production privacy policy for a live financial services platform.

A production version connected to forms, authentication, CRM, analytics, APIs, widgets, or funding systems will need a reviewed privacy policy that matches the actual data flows.

Browser localStorage

The static dashboard may use `localStorage` to save demo state in your browser. This can include partner profile demo fields, sample leads, notes, resource favorites, training progress, attribution parameters, theme preferences, and local analytics events.

  • Partner workspace data stays in your browser unless you export or share it.
  • Clearing browser data may delete partner workspace records.
  • Other users of the same browser profile may be able to see partner workspace data.
  • Do not treat localStorage as secure storage.

Information you should not enter

Do not enter sensitive personal, business, borrower, banking, tax, or credential data into the partner workspace.

  • No Social Security numbers.
  • No full tax IDs or EIN records beyond harmless placeholders.
  • No bank account numbers.
  • No bank login credentials.
  • No full loan applications.
  • No tax returns, statements, or sensitive documents.
  • No private borrower files.

Partner attribution

The static site may capture URL parameters such as partner ID, referral ID, affiliate ID, source, or UTM parameters and store them locally. This is for demo attribution and link-building behavior only.

Static attribution does not create a live referral, payable commission, account record, or CRM record.

Local analytics demo

The static script layer may record local browser-only page view and click events in localStorage. This is not connected to an external analytics provider in the static version.

Future production analytics should disclose what is collected, why it is collected, how long it is retained, and which vendors process it.

Future backend requirements

Before this system handles live partner or lead data, operators should define and review:

  • Data collection purposes.
  • Required consent and authorization language.
  • Authentication and access controls.
  • CRM and vendor data flows.
  • Data retention and deletion process.
  • Audit logging.
  • Security controls and incident response.
  • Production privacy policy and terms.

Questions

For now, treat this repo as a static workflow prototype. If a live deployment starts collecting real data, replace or supplement this page with a production-ready privacy policy that matches the actual business, tools, vendors, and legal requirements.
